折叠面板 Accordion
传统 YOLO（v1–v7）使用预设锚框（Anchor）作为先验框，需要对每张图像与锚框计算 IoU 进行匹配。Anchor-free 方法（YOLOv8/v10）直接预测目标中心点相对 Grid Cell 的偏移量和宽高，无需锚框先验，简化超参调节，在小目标上泛化更好。
YOLOv4 是 YOLO 系列技术密度最高的版本，主要创新包括：Mosaic 数据增强（4图拼接扩充上下文）、CIoU Loss（考虑长宽比一致性的边界框回归损失）、Mish 激活函数、DropBlock 正则化、CSPNet（跨阶段特征复用减少重复梯度）以及 PANet 颈部网络（双向特征融合）。
FPN（Feature Pyramid Network）自顶向下逐层融合高层语义特征，增强小目标的语义信息。PAN（Path Aggregation Network）在 FPN 的基础上再增加一条自底向上的路径，将低层高分辨率的定位信息向上传递，弥补 FPN 中位置信息的损失。YOLOv4 起使用 FPN + PAN 双向融合，兼顾语义与定位。
非极大值抑制（NMS）是一个串行的后处理步骤，无法并行化，在目标密集场景（如行人检测）下延迟尤为突出，且 IoU 阈值需要人工调节。YOLOv10 引入双重标签分配策略（Dual-label assignment）：训练时使用一对多分配获得丰富监督信号，推理时切换为一对一分配直接输出唯一预测框，从根本上消除 NMS，实现真正的端到端检测。
嵌套标签页 Nested Tabs
骨干网络演化

Darknet-24 v1 (2015) → Darknet-19 v2 → Darknet-53 v3 (ResNet残差块) → CSPDarknet-53 v4/v5 (跨阶段融合，减少重复计算) → E-ELAN v7 (高效层聚合，扩展混洗) → C2f v8 (C3升级版，梯度流更丰富) → CSPNet-v10 v10 (紧凑倒置块减参数)
关键训练策略

数据增强：Mosaic (v4+) · CutMix · MixUp · Copy-Paste · 随机仿射变换 · HSV 颜色抖动

正则化：DropBlock · Label Smoothing · Close Mosaic (最后10epoch关闭Mosaic)

优化器：SGD + 余弦退火学习率 · AdamW (v8) · EMA 指数滑动平均权重

损失函数：MSE → IoU → GIoU → CIoU (v4+) → DFLoss + VFLoss (v8+)
部署格式支持

.pt PyTorch 原生，调试首选  |  .onnx 跨框架通用格式  |  TensorRT NVIDIA GPU 极速推理  |  CoreML Apple 设备端推理  |  OpenVINO Intel 硬件优化

一般流程：训练 → 导出 ONNX → 量化（INT8/FP16）→ TensorRT 编译 → 部署
推理完整流程

1. 预处理：Letterbox 等比缩放至 640×640，归一化 [0,1]，BGR→RGB，增加 Batch 维

2. 前向推理：模型输出原始张量，形状为 [B, 84, 8400]（COCO 80类 + 4坐标 × 8400锚点）

3. 后处理：解码坐标（xywh→xyxy）→ 置信度过滤（conf>0.25）→ NMS（YOLOv10略去此步）

4. 反缩放：将预测坐标从 640×640 空间映射回原图尺寸

5. 可视化：绘制边界框、类别标签、置信度分数

发展时间轴 Timeline
2015 · CVPR · Redmon, Farhadi et al.
YOLOv1 — 单阶段检测革命
首次将目标检测统一为单次回归问题。45 FPS · VOC mAP 63.4 · Darknet-24 骨干 · 将检测速度提升至实时级别。
2016 · CVPR · Redmon, Farhadi
YOLOv2 / YOLO9000 — 锚框引入
引入 Anchor Boxes、Batch Normalization、高分辨率分类器预训练。YOLO9000 可识别 9000+ 类别（WordTree 联合训练）。
2018 · arXiv · Redmon, Farhadi
YOLOv3 — 多尺度特征检测
引入 Darknet-53 骨干（含残差块）、FPN 多尺度输出（3个尺度）、Logistic 代替 Softmax 支持多标签。COCO mAP 33.0。
2020 · arXiv · Bochkovskiy, Wang, Liao
YOLOv4 — 技术集大成
CSPDarknet + PANet + Mosaic增强 + CIoU Loss + Mish激活 · COCO 43.5 mAP · 65 FPS on V100 · 确立现代 YOLO 基础架构范式。
2020 · GitHub · Ultralytics (Glenn Jocher)
YOLOv5 — 工程化与生态建设
PyTorch 原生重写，自动混合精度、AutoAnchor、超参数进化优化。COCO mAP 45.4 · 112 FPS · 生态最完整、社区最活跃。
2022 · CVPR · Wang, Bochkovskiy, Liao
YOLOv7 — 速度精度双 SOTA
E-ELAN 扩展高效层聚合网络 + 辅助头训练 + 再参数化。161 FPS · COCO mAP 51.2 · 发布时在速度精度帕累托前沿全面领先。
2023 · Ultralytics
YOLOv8 — Anchor-free 统一框架
C2f 模块替代 C3、Anchor-free 检测头、DFL 分布焦点损失。统一支持检测/分割/姿态/分类四任务 · COCO mAP 50.2 · 成为新一代基准。
2024 · 清华大学 Wang et al.
YOLOv10 — NMS-free 端到端检测
双重标签分配彻底消除 NMS 后处理 · 紧凑倒置块降低参数量 · YOLOv10-X COCO 54.4 mAP · 开启 YOLO 端到端新时代。
